## Exercice 6 — Pipeline ETL déclenchée par l’arrivée d’un fichier généré par un autre DAG (niveau avancé)

### Objectif pédagogique

Dans cet exercice, mettre en place une chaîne de traitement complète dans Airflow avec deux DAGs :

1. un **DAG producteur** qui génère un fichier dans un répertoire cible
2. un **DAG ETL consommateur** qui attend l’arrivée de ce fichier avant de démarrer son pipeline

L’objectif est de manipuler :

- la création de DAGs
- les dépendances entre tâches
- l’attente d’un fichier avec un `FileSensor`
- les logs Airflow
- les états des tâches
- l’orchestration d’un pipeline ETL simple

---

### Contexte

Dans un contexte d’ordonnancement, il est fréquent qu’un traitement ne puisse démarrer que lorsqu’un fichier est disponible.

Exemple métier :

- un premier traitement dépose un fichier d’entrée
- un second traitement attend ce fichier
- dès que le fichier est disponible, il lance une chaîne ETL :
  - extraction
  - transformation
  - chargement
  - génération d’un rapport

Dans cet exercice, le fichier sera généré non pas manuellement, mais par un **autre DAG Airflow**.

---

### Résultat attendu

À la fin de l’exercice, on doit obtenir :

- un DAG `generate_input_file`
- un DAG `etl_on_file_arrival`
- un fichier déposé dans un répertoire partagé
- un `FileSensor` qui attend ce fichier
- un pipeline ETL exécuté après détection du fichier

---

### Arborescence logique de l’exercice

```text
DAG 1 : generate_input_file
    └── create_file

DAG 2 : etl_on_file_arrival
    ├── wait_for_file
    ├── extract
    ├── transform
    ├── load
    └── generate_report
```
---

Le fichier qui doit être créé se fera dans le répertoire suivant :  **/tmp/airflow_lab/input/**

Voici la commande bash complète que vous devez lancer pour la création du fichier à surveiller : 

```python
bash_command="""
        mkdir -p /tmp/airflow_lab/input
        cat > /tmp/airflow_lab/input/customers.csv <<EOF
id,name,city
1,Alice,Paris
2,Radia,Lyon
3,Charlie,Marseille
4,Ahmed,Chambéry
EOF
        echo "Fichier généré avec succès"
        """
``` 
A vous d'introduire cette commande au niveau de votre dag.
